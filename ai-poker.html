AI Poker Bot (Spring)

Build a poker-playing AI that can read the table, manage risk, and bluff when it counts. Teams will design agents that make betting decisions under uncertainty, then compete in a campus tournament to crown the smartest bot.

Key Info

  • Format: Team-based AI agent tournament (Texas Hold’em–style; exact variant announced with the rules).
  • Skill focus: Reinforcement learning, search/heuristics, game theory, state representation, and evaluation.
  • Dates: Spring — final schedule TBA. (Sign up to get the announcement.)

How It Works

  1. Form a team (or sign up solo and we’ll match you).
  2. Download the starter environment (engine + API + evaluation harness).
  3. Implement your bot: choose heuristics, RL, CFR-style approaches, or hybrids.
  4. Test via local round-robins and weekly scrimmages; submit your final agent for the tournament.

Rules & Scoring (preview)

  • Sandboxed agents: Bots interact only through the provided API (no external I/O during matches).
  • Deterministic builds: Submit a reproducible package (requirements + seed handling).
  • Tournament scoring: Points by table placement across rounds; top teams advance to finals.
  • Fair play: No collusion; no adversarial system exploitation.

Full rules, APIs, and starter code release with the spring announcement.

Suggested Tech Stack

  • Python (recommended) or C++/Java for performance-critical agents.
  • Popular libs: NumPy, PyTorch (if using RL), JAX or TensorFlow (optional).
  • Version control on GitHub; use branches and PRs for experiments.
